This page is dedicated to informing people how to get and maintain privacy within our not so private online spaces.

Websites

Now, its no secret that alot of websites often include crafty tricks in order to limit the user's liberties within the site, aswell as track every single datapoint that the user happens to expose about themselves. But, if this is so commonly the case, then how do people know which websites are safe?

Put simply, you dont!

There is ultimately no way to tell if a website or a website hoster is tracking you or not. In an attempt to make this site more trust worthy, I have made the site completely static, which means that it does not contain any images or javascript, which are two things that are very commonly used as data collection points. However, not even this site is 100% safe!

There is a possibility that either 1. I am a bad actor and im just saying that the source repository for this site is actually being used, when its not, 2. Theres a chance that the website hoster i use, Vercel, has made unauthorised changes to this site, 3. A completely independent bad actor has tampered with this site, or even 4. That your ISP has tampered with your ability to view the original version of this site.

Now, for the middle 2 cases, there isnt a whole lot you as the user can do. However, the first and the fourth are exceptions. Since this site is pure HTML, you can use an image and JS blocker, just incase, and then also manually verify the HTML script within the developer menu, which should give you a pretty good idea of whether I have maliciously tampered with the site.

The fourth is also an exception. If you EVER see a browser give you a warning before visitng this site, DO NOT enter the site until the warning is no longer present. This is because this site uses HTTPS, which means that for an ISP to change the website, they would have to invalidate this HTTPS protocol, which is what throws the error at the user.
